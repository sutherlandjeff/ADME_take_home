# Instructions for Candidates

Thank you for taking the time to complete this exercise. It is designed to reflect real work you would do in this role.

---

## What we're looking for

- Clean, readable Python and Django code
- Understanding of Django's ORM, views, and template system
- Thoughtful test coverage
- Clear git history with meaningful commit messages
- Good judgment about what to implement vs what to leave for later

We are **not** looking for a pixel-perfect UI or a complete feature set. Working code with clear reasoning matters more than covering every TODO item.

---

## Setup

See `README.md` for full setup instructions.

---

## Your workflow

1. **Fork this repository** to your own GitHub account
2. Clone your fork locally
3. Create a feature branch:
   ```
   git checkout -b feature/your-name
   ```
4. Work through the tasks in `TODO.md`
5. Commit regularly with clear messages, e.g.:
   ```
   git commit -m "Add status filter to lab dashboard"
   git commit -m "Add barcode uniqueness validation with tests"
   ```
6. Push your branch and **open a Pull Request** against `main` in your fork

Then share the PR URL with us.

---

## Time limit

We suggest spending **3–4 hours**. If you go over, that's fine — but please note how long you spent in your PR description.

If you run out of time before finishing, include a brief note in your PR or in a code comment explaining what you would have done next.

---

## PR description

When you open the PR, please include:

- What you implemented
- Any design decisions or trade-offs you made
- What you would improve or add given more time
- Approximately how long you spent

---

## Questions

If anything is ambiguous, use your judgement and note your assumption in the PR description. There is no single "right" answer — we're interested in how you think through the problem.
